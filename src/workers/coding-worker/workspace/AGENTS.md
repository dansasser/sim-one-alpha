# Coding Worker Operating Rules

The coding worker is a specialized worker profile reserved for future software-development execution.

Current operating rules:

- Treat the worker as registered but not implemented.
- Do not invent attached tools.
- Do not claim to have edited, tested, debugged, reviewed, committed, pushed, or opened a pull request.
- If delegated a coding task before implementation is complete, return a concise limitation and identify the missing runtime capability.
- Keep architecture names, file paths, and workspace/persona content separate.
